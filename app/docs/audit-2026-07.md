# Auditoría técnica y visual — julio de 2026

## Alcance

Auditoría del commit base `b439a84` y remediación en `codex/email-builder-overhaul`. El análisis cubrió seguridad, datos, APIs, renderizado de email, accesibilidad, responsive, activos visuales, dependencias y pruebas. No se modificaron Supabase, Vercel ni GitHub remotos.

## Hallazgos del baseline

| Severidad | Hallazgo | Riesgo observado | Remediación |

| Alta | Upload confiaba en extensión/declaración del cliente | Archivos falsos o excesivos | Límite de 8 MB, decodificación real con Sharp, MIME permitido y normalización |

## Arquitectura resultante

- Autenticación por magic link PKCE con `shouldCreateUser: false`; no existe registro público.
- Clientes Supabase independientes para navegador, servidor y administración. El service role solo aparece en scripts operativos de migración/seed.
- `EmailDocumentV3` separa asunto, preheader, locale, tema, bloques y cumplimiento. Los drafts/enlaces legacy se adaptan sin romper su lectura.
- Renderer único con ancho base de 600 px, tablas, CTA VML, fallback de fondos, stacking móvil, preheader oculto y footer GHL.
- Tokens permitidos: `{{unsubscribe}}`, `{{location.full_address}}` y merge fields simples de contacto. El texto restante se escapa antes de renderizar.
- Assets con ID, marca, tipo, alt, dimensiones, MIME, peso, variante, autor, fecha, prompt y uso previsto.

## Sistema visual

La interfaz usa navegación azul marino de AD Media y superficies claras/neutras para edición, con preview blanco. Se añadieron navegación móvil off-canvas, toolbar sticky, objetivos táctiles de 44 px, foco visible, reduced motion y previews limitados al viewport.

El starter pack contiene, para AD Media y AMO, dos heroes JPEG 1200×600 y dos adsets coordinados de tres tiles JPEG 800×800. Cada imagen de email tiene thumbnail WebP; los prompts, alt text y usos están en `assets/starter-pack.json` y `public/email-assets/manifest.json`. Los logos e iconos tienen maestros SVG y PNG 48/96 generados de forma determinista.

## Verificación automatizada

Los comandos de aceptación son:

```bash
npm ci
npm run lint
npm run typecheck
npm test
npm run test:e2e
npm run build
npm audit
```

Vitest cubre sanitización, protocolos, merge fields, migración legacy, schemas, exportación y las 18 combinaciones de 9 plantillas × 2 marcas. Playwright cubre login, dashboard, editor, modal de marca y viewports 390/768/1024/1440, incluyendo ausencia de scroll horizontal y snapshots visuales.

## Validación manual pendiente de entorno live

La matriz final debe ejecutarse desde GHL con cuentas reales: Gmail web/móvil, Apple Mail, Outlook clásico/365/web y Yahoo; verificar mobile, dark mode, VML, imágenes remotas, alt text y resolución de merge fields. Es una validación de despliegue, no una carencia de la suite local.
